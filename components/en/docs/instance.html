Instance

When we import Bouer it will added a function to the global object, so to begin using it we need to create a new instance of Bouer Class. This constructor expects two parameters: 


        var app = new Bouer(element, options);

Element

The element is the mandatory parameter that represents the section that the instance will control in the application, it must be passed as a String in the format of a CSS Selector. Example: #app, .main-app or body.

Options

Options is the optional parameter, which can be used to set initial data, app configurations, components (we'll talk about this later), etc. If passed, it must be an object containing all the information we need when the instance is created.

Properties

As previously stated, in the options parameter several properties can be defined which are: 


        var app = new Bouer('selector', {
          /** The data of the application, havig the properties and methods */
          data: Object,

          /** The data that cross scopes (New Scope or Components Scope)   */
          globalData: Object,

          /** The configuration of the instance */
          config: Object,

          /** The components of the application */
          components: Component[],

          /** The custom directives for the application */
          directives: { [key: string]: Directive },

          /** The dependencies of the instance */
          dependencies: { [key: string]: Object },

          /** Appends delimiters to the instance */
          delimiters: Delimiter[],

          /** Middlewares that will be used in the application */
          middleware: (
            function configure (directive: string, configureCallback: MiddlewareConfig) { },
            app: Bouer
          ) { },

          /** The hook that will be called before the component is loaded */
          beforeLoad(event: CustomEvent) { },

          /** The hook that will be called after the component is loaded (Compiled) */
          loaded(event: CustomEvent) { },

          /** The hook that will be called before the component is destroyed */
          beforeDestroy(event: CustomEvent) { },

          /** The hook that will be called after the component is destroyed */
          destroyed(event: CustomEvent) { },
        });

Data

As already mentioned, the data property allows you to set the data that the application will have when it starts, however there is a way to add data while it is already running with the app.set({ ... }) method (We'll talk about that in the methods section).
Data or properties defined on data are not normal properties, they are reactive, this means that they react to changes made to them. 


        

          
 {{ message }} 

        

          

        var app1 = new Bouer('#app1', {
          data: {
            message: 'Hello World!'
          }
        });

        app1.set({ version: '3.0.0' });

Is in data property where the functions need to be defined too, and they do not respect scopes (we'll talk about this later), they can be accessed even if a scope is created.

Dependencies

This property allow to add dependencies in the current instance... All the dependencies can be accessed at app.dependencies object, which are defined providing a [key: string] and The dependency object. { 'my-dependency': {...} } 


      var app = new Bouer('#app', {
        dependencies: {
          'html-parser': HTMLParser
        }
      });

      var parser = app.dependencies['html-parser'];

We can create an instance without a variable, meaning that the instance cannot be accessed by the browser console.
