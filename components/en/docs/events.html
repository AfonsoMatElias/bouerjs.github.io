Events

We can use on: to listen for DOM events and run some javascript code when it is dispatched. In some examples above we use the click event so we imagine that you must be a little familiar with it.

Bouer events can be categorized into 3 types, which are: DOM Standard, Bouer and Custom events. For all of them, they can be heard with the following syntax: on:eventname="function/expression".

When the events are dispatched, the event object can be accessed in the event variable.

Standard Events

These are DOM Standard Events, such as onclick, ondblclick, onkeyup, etc.

Bouer Events

These are Bouer's internal events, which we can mention:

  • compile – this event is dispatched when the element containing the event is fully compiled.
  • add – this event is dispatched when an element of the e-for directive is inserted to the DOM. It works with e-for and e-req.
  • request – this event is dispatched when a request made using a e-req directive.
  • response – this event is dispatched when we get the response using a e-req directive.
  • fail – this event is dispatched when a request fails using a e-req directive.
  • done – this event is dispatched when a request is finished.

Custom Events

These are the events that we can freely set the names according to our liking. Do not use reserved names, which are event names mentioned above. To emit these events we use the app.emit(eventName: string, options?: object) method.

Modifiers

To add modifiers to the events use . (dot) after the eventName. The sintax is on:eventName .modifier1.modifier2="expression"

The following modifiers are available:

  • prevent – calls event.preventDefault() before running the handler.
  • stop – calls event.stopPropagation(), preventing the event reaching the next element.
  • once – remove the event after the first time call.

Modifiers can be chained together, for example.: on:click.stopPropagation.once="doSomeThing"
