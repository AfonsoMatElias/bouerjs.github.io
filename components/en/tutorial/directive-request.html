Notice that the syntax is very similar to e-for, meaning that we can also user filters as e-for does.

When we are using e-req, we need to configure it with middleware to be able to make external requests, the middleware configuration has the following syntax: 


    var app = new Bouer('selector', {
      middleware: function (configure, app) {
        // Configuring the interceptions of the `request`
        configure('req', function (onBind, onUpdate) {

          onBind(function (context) {
            // TODO: Your after bind logic code here...
            // Always `return` an object with `data` property
            // return { data: [] | {}, ... };
          });

          onUpdate(function (context) {
            // TODO: Your update logic code here...
            // Always `return` an object with `data` property
            // return { data: [] | {}, ... };
          });

        });
      }
    });

Using e-req with of request type, we tell Bouer to expect a list of items. If we want to request an object we need to use as request type, so we tell Bouer that we expect an object.

For further information about e-req, check the documentation on Directive e-req.
