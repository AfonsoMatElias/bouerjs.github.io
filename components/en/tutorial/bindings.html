Bindings (Data Rendering)

The previous chapter we rendered data as a simple One Way Data Binding, but we can also render as Two Way Data Binding. 


    
 Image Name

We can define Two Way Data Binding is a way to link an Element to a Property, listening to changes in in both sides.

In Bouer we can achieve Two Way Data Binding using the directive e-bind:attribute="property". By default it binds the main attribute of the element, in case of input[type="text"] it binds value attribute and in input[type="checkbox"] it binds the checked attribute, so we can define it like this e-bind="property".

By the default the binding model is the value attribute, to be able to define a model for the binding we use the :value="{ /* ... */ }" attribute.

We also can bind a list of items easily, controlling addition and removal in very simple way. More details of Bindings.
