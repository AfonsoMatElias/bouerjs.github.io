In Bouer all the properties and methods are defined in options.data and options.globalData. 


    var app = new Bouer('selector', {
      data: { /* ... */ },
      globalData: { /* ... */ }
    })

The data properties are only visible in the first scope, meaning that if we create a new scope we won't be able to access the data properties.

Unlike data, globalData has different behavior, all the properties stays visible independent of the scope.

One of the simple usecase can be: 


    var app = new Bouer('#app', {
      globalData: {
        user: {
          name: 'AfonsoMatElias',
          profile: 'Admin'
        }
      }
    });

We create a new scope by setting the data directive to the element that we want to be compiled with the new scope: 


     Content

Inside of data expression we are able to access the current scope with $data variable, and $root variable to access the application instance data.

To be able to add properties to the application instance we use: app.set({ /* ... */ }). 


    var app = new Bouer('selector', {
      loaded: function () {

        this.set({ /* ... */ });

      }
    });
