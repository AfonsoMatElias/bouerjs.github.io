Data Rendering

The template it self just render static content as normal HTML, to be able to create a Bouer App and render some data, we neeed to create a new instance of Bouer providing the application selector as the first parameter and add in options.data all the properties we want to be rendered. 


    var app = new Bouer('selector', {
      property: DataType
    });

We can also add data properties using the app.set({ /* ... */ }) method, allowing us to set it before or after the app is fully loaded.

Using delimiters like {{ expression }} we are able to render javascript code, but it needs to be be an expression that returns value. 


    
 {{ expression }}

But we can also render Element text content using e-text directive, this wait it seems a little bit cleaner.

Dynamic Attributes

If we don't want to render data this way, exposing the delimiter in the actual element attribute, we can still use the directive e-attribute="{{ expression }}", it creates the attribute after being compiled. 


    
 Title

For toggleable attribute creation (if it does not exists), we just define ao Object Literal the expression without delimiters e-attribute="{ 'value-to-toggle' : condition }". If attribute value is empty the attribute is automatically removed.
